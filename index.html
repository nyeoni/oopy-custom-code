TL;DR

자바스크립트 개발자라면 꼭 알아야하는 개념인 호이스팅..!! 맨 처음 호이스팅에 대한 개념을 듣고 도대체!!! 왜!!! 호이스팅이 일어나는지, 장점이 뭔지가 궁금했다. 이에 대해 구글에서 서칭을 한 결과, 호이스팅의 큰 장점은 없고 그저 자바스크립트을 인터프리터 언어로 설계할때 도입되었던 개념이다(허무…., 혹시 아는 사람 있으면 댓글 남겨주세욧..!!).

호이스팅

Javascript 에서 호이스팅(hoisting) 이란, 인터프리터가 변수와 함수의 메모리 공간을 선언 전에 미리 할당하는 것을 의미한다.
•
자바스크립트 엔진은 소스코드를 실행하기에 앞서 소스코드의 평가 과정을 거친다.
◦
소스코드의 평가과정에서 변수 선언을 포함한 모든 선언문들을 먼저 실행한다.
◦
소스코드 평가 과정이 끝나면 변수 선언문을 제외한 소스코드를 한 줄 씩 순차적으로 실행한다.
•
변수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트 고유의 특징을 호이스팅 이라고 부른다.
•
변수 선언 뿐만 아니라, var, let, const, function, function *, class 키워드를 사용해서 선언하는 모든 식별자(변수, 함수, 클래스 등)은 호이스팅 된다.
→ 모든 선언문은 런타임 이전 단계(평가단계) 에서 실행되기 때문
위의 내용에 따른다면, 모든 변수 및 선언문들은 호이스팅 이 된다. 그러나!!! 호이스팅이 되지 않은 것처럼 동작하는 키워드들이 있다. 그건 바로 let 과 const 이다. 이들은 ES6 문법에 새로 추가된 키워드들로 기존의 var 변수의 단점을 보완하여 만들어진 새로운 변수 선언 키워드 들이다.

호이스팅이 안된 것 처럼 동작하는 키워드

호이스팅이 안된 것 처럼 동작하는 키워드들은 바로 let, const, class 이다.
•
let, const, class 는 호이스팅이 되어 선언문 실행되지만 초기화는 되지 않는다.
•
초기화가 실행되지 않는다는 것은 메모리 할당이 안된다는 뜻이다. 즉, 선언문을 통해 스코프의 변수 객체에는 등록되지만 초기화는 하지 않아 메모리를 할당받지 않는다.
•
런타임 이전에 선언 단계 만 실행되고 초기화 단계 는 런타임에서 변수 선언문에 도달했을 때 실행된다.
•
초기화 단계 실행 이전에 참조하려하면, 참조 에러 (Reference Error) 가 발생된다.
•
이 때, 변수를 참조할 수 없는 구간을 일시적 사각지대 (TDZ) 라고 부른다.

호이스팅이 의심의 여지 없이 잘~ 동작하는 키워드

var, function, import 구문이다.

1. var

•
var 키워드 변수는 선언 단계 와 초기화 단계 동시에 진행된다.
•
자바스크립트 엔진의 평가 단계에서 변수 객체에 등록하는 동시에 메모리를 undefined 로 할당한다.
•
그렇기 때문에 변수 값이 할당되기 전에 호출해도 참조 에러가 발생하지 않는다.

2. function

•
함수 선언식의 경우, 선언/초기화/할당 단계가 모두 진행된다.
•
그래서 할당 전 호출해도 undefined 가 아니라 함수 내용이 리턴된다.
•
중요한 포인트!) 함수 표현식이 아닌, 함수 선언식이 호이스팅 된다는 점

변수의 생성 단계

자바스크립트 변수는 선언 -> 초기화 -> 할당 이라는 3가지 단계를 거쳐 생성된다.

선언 단계 (Declaration phase)

•
변수를 실행 컨텍스트 변수 객체 에 등록하는 단계
•
이 변수 객체는 스코프가 참고하는 대상이 된다.

초기화 단계 (Initialization phase)

•
실행 컨텍스트에 존재하는 변수 객체 에 선언 단계에서 생성된 변수를 위한 메모리를 만드는 단계이다.
•
이 단계에서 할당된 메모리에 undefined 를 할당한다.

할당 단계 (Assignment phase)

•
초기화 단계에서 undefined 로 초기화된 메모리에 다른 값을 할당하는 단계이다.
위에서 사용된 실행컨텍스트와 스코프에 대한 개념을 처음 듣는다면, 일단 변수는 선언 → 초기화 → 할당 이라는 세 단계를 거쳐 생성된다는 사실만 알고 넘어가도록 하자. 실행 컨텍스트와 스코프에 대해 추가적으로 궁금하다면 실행컨텍스트와 스코프 포스팅 참고할것

TDZ (Temporal Dead Zone)

스코프 시작 지점부터 초기화 단계 시작 지점까지 변수를 참조할 수 없는 구간을 일컫는다. 다른 말로 일시적 사각지대 라고도 한다.
•
TDZ 는 let, const, class 등의 구문의 유효성을 관리한다.

let, const, class

위에서 언급했듯이, let, const, class 는 TDZ 전에는 변수를 사용할 수 없다.
/////////////////// a; // b; // new C('c'); // /////////////////// /////// TDZ /////// /////////////////// /////////////////// let a = 1; const b = 2; class C { constructor(id) { this.id = id; } }
JavaScript

constructor() 내부의 super()

부모 클래스를 상속받았다면, 생성자 안에서 super() 를 호출하기 전까지 this 바인딩은 TDZ 에 있다.
class MuscleCar extends Car { constructor(color, power) { this.power = power; super(color); } } // Does not work! const myCar = new MuscleCar(‘blue’, ‘300HP’); // `ReferenceError`
JavaScript

기본 함수 매개변수

기본 매개변수는 글로벌과 함수 스코프 사이의 중간 스코프에 위치한다. 기본 매개변수 또한 TDZ 제한이 있다.
const a = 2; function square(a = a) { return a * a; } // Does not work! square(); // throws `ReferenceError`
JavaScript
•
기본 매개변수 a는 선언 전에 a = a표현식의 오른쪽에서 사용되었다. a에서 참조 에러가 발생한다.

결론

•
var 로 선언한 변수의 경우, 호이스팅 시 undefined 로 변수를 초기화한다.
•
let, const 로 선언한 변수의 경우, 호이스팅 시 변수를 초기화 하지 않는다.
•
function 선언문은 선언, 초기화, 할당단계까지 호이스팅 된다.
